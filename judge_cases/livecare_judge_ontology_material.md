# 라이브케어 협상 시뮬레이션 — Judge 온톨로지 개발 재료
### 대화 전문 + 차원별 매칭 + Context · 3개 케이스 (v1.0)

> **수신**: AI 박사님 (Judge 온톨로지·학습 데이터 설계)
> **목적**: 판매자 라이브케어(소 체내 IoT 바이오캡슐, 데이터 서비스)를 3-track(직접/유통/공공)으로 협상 시뮬레이션한 결과를 Judge 7차원에 매핑. **축산 IoT/데이터 도메인**에서 대체재 극단·채널 긴장·ODA 공공 축을 검증하는 재료.
> **연계**: 차원 정의 = 작성가이드 §2·기획서 7.2 / 협상 = 7-A / 가설 카드 = 7-B·스키마 v1.1.
> **판매자 프로필** `[관찰]`: 소 체내 삽입 IoT 바이오캡슐 → 생체 데이터 → AI 분석(발정·배란·질병·분만·운영). 포지셔닝 = "제품이 아닌 데이터 서비스". Key Benefit = 상업(Problem Solving+Cost Reduction) / ODA·공공(Impact+Problem Solving). 3-track = 직접 대형농장 / 현지 ICT·장비 유통사 / 정부·ODA.

## ⚠️ 0. 유효 범위와 한계

**온톨로지 설계도이지 학습 벽돌 아님.** 구매측 `[구성]`, 정답 앵커 없음. 가치 = **축산 IoT 도메인에서 3-track이 정합 기준을 어떻게 3분화하는지 + 대체재 극단·채널 긴장·공공** 형판 확보.

**채운 빈 축(앞선 사에 없던 것)**: 축산 IoT/데이터 · **대체재 비교가 딜 전체를 지배**(기존 목걸이형 vs 체내 삽입형) · 진입 채널 긴장(직접 vs 대리점 관행, 양방향) · ODA/공공 트랙(Impact+PS) · 현지 실행 파트너 역량 검증(#04) · 한 판매자의 정합 기준 3분화.

# 케이스 L1 — 홋카이도 낙농법인 유키미르크 (Track A 직접) · out/sell/conditional

**🔒 봉인**: 사육 ~2,800두 대형 목장. 구매이유 = 인력난·수의사 부족으로 개체관리 자동화(발정 놓침 공태일 손실이 실제 비용). 대체재(핵심) = 이미 목걸이형(외부 부착) 일부 도입 — "체내형이 목걸이형보다 뭐가 나은가"가 결정적. 신뢰임계 = 체내 삽입에 대한 수의사·조합 보수성 + 일본 인증 + 시술·회수 부담. 채널 긴장 = 낙농 장비를 대리점 경유로 사는 관행 강함 → 직접거래(Track A) 저항.

## 대화 전문 (context)

**R1** 🟦 라이브케어: "소 체내 삽입형 IoT 바이오캡슐로 발정·질병·분만을 실시간 관리하는 라이브케어입니다. 저희는 제품이 아니라 데이터 서비스를 제공해 인력난 속에서도 개체 관리를 자동화하고 공태일 손실을 줄여드립니다. 15분 소개 가능하실까요?"
🟥 유키미르크: "개체 관리 자동화는 절실합니다 — 사람도 수의사도 부족하니까요. 그런데 솔직히, 저희는 이미 목걸이형 모니터링을 일부 쓰고 있어요. 발정 감지 같은 건 그걸로도 어느 정도 됩니다. 체내에 넣는 건 수의사들도 조합도 조심스러워하고요 — 시술도 해야 하고 문제 생기면 회수도 어렵잖아요. 목걸이형 대비 체내형이 정말 나은 점이 뭔지, 삽입 부담을 감수할 만한지가 궁금합니다."

**R2** 🟦 라이브케어: "정확한 질문입니다 — 목걸이형 대비 우위가 삽입 부담을 넘어야 한다는 거죠. 핵심 차이는 데이터의 질과 범위입니다. 목걸이형은 활동량 중심이라 발정 추정 정확도에 한계가 있고 질병·분만·체온 같은 체내 신호는 못 잡습니다. 저희 체내 캡슐은 반추위 환경에서 체온·활동을 직접 측정해 발정뿐 아니라 질병 조기경보·분만 예측까지 정확도 높게 제공합니다 — 목걸이형이 못 푸는 문제를 풉니다. 삽입 부담은 이해합니다. 그래서 소수 개체로 PoC를 먼저 제안합니다 — 목걸이형과 나란히 두고 정확도를 직접 비교하실 수 있게요. 15분 미팅에서 비교 데이터와 PoC 설계를 보실까요?"
🟥 유키미르크: "목걸이형과 나란히 비교하는 PoC라면 판단 근거가 생기겠네요. 질병·분만까지 잡는다면 확실히 다른 얘기고요. 다만 두 가지 — 일본 내에서 이 캡슐이 인증이나 검증된 사례가 있나요? 그리고 저희는 보통 이런 장비를 대리점을 통해 들이는데 직접 거래는 낯설어요. AS나 문제 생겼을 때 대응이 걱정됩니다."

**R3** 🟦 라이브케어: "일본 내 검증은 — 이번 PoC 자체가 홋카이도 환경 첫 검증 레퍼런스가 되고, 인증은 [보유 상태/진행 계획]을 미팅에서 투명하게 공유하겠습니다. AS·대응 우려는 — 저희가 직접 초기 밀착 지원을 하되 확산 단계에서는 현지 축산 장비 파트너와 협력하는 방안도 열려 있습니다(직접 거래 신뢰가 쌓이면). 우선 소수 개체 PoC로 목걸이형과 정확도를 비교하면서 대응 체계도 함께 검증하시죠. 다음 주 15~30분 PoC 설계와 비교 방법을 잡을까요?"
🟥 유키미르크: "PoC로 비교하면서 대응도 같이 본다면 시작해볼 만하네요. 목걸이형과 나란히 비교하는 조건으로 소수 개체부터 해보죠. 인증 상태는 미팅에서 꼭 확인하고요."

**■ 종료: 비교 PoC 합의(conditional).** 대체재 우위(질병·분만)를 PoC 비교로 실증, 채널 긴장은 '직접+파트너 협력'으로 완화, 인증은 조건.

# 케이스 L2 — 메콩 아그리테크 (베트남 축산 ICT 유통사, Track B) · out/sell/conditional

**🔒 봉인**: 베트남 축산 ICT·장비 유통사, 현지 대형 농장·협동조합 채널. 구매이유(채널 파트너로서) = 자사가 팔아 마진을 남길 수 있는가(최종 사용자 아닌 재판매자). 관심 = 베트남 낙농·양돈 성장, 차별화 포트폴리오. 우려 = 현지 농가 지불능력 낮음, AS·기술지원 체계, 라이브케어 직접(Track A) 진출 시 자기 채널 위협 → 채널 보호. Willingness 높음(신제품 원함) — 단 마진·독점·지원이 관건.

## 대화 전문 (context)

🟥 메콩: "라이브케어 제품에 관심 있습니다. 베트남 축산 시장이 빠르게 크고 있고 저희가 대형 농장·협동조합에 강한 채널을 갖고 있습니다. 저희를 통해 들어오시면 시장 진입이 빠를 겁니다."

🔎 [핵심—채널 파트너는 최종 구매자와 판단 기준이 다르다]: 메콩은 캡슐을 자기가 쓸 게 아니라 되팔 유통사. 정합·보완성을 "농장 문제를 푸는가"가 아니라 "우리가 이걸 채널로 팔아 마진을 남기는가"로 판단. 키뮤 1-B(광고대행사=중간자)와 같은 구조 — "구매자의 구매자가 누구냐"(여기선 농장). 정합이 두 겹: 메콩이 팔 수 있어야(1차) + 농장이 사야(2차).
🔎 [대체재/기회비용—메콩 관점]: 메콩은 다른 ICT 제품도 취급 가능 → 라이브케어에 리소스 쓰는 게 남는가. 라이브케어는 차별화+마진+지원을 줘야 채널 우선순위 획득.
🔎 [채널 긴장—Track A vs B, 거울]: L1(일본)에선 농장이 "직접거래 낯설다" 했는데 여기선 유통사가 "라이브케어가 직접 갈까 봐" 우려 = 같은 채널 긴장의 반대편. 메콩은 채널 보호 위해 독점·지역 보장 원할 수 있음(코봇 C2 유사).
🔎 [현지 실행—#04 교훈]: 베트남 현지 지불능력·AS·기술지원이 관건. #04의 "신뢰할 현지 실행 파트너"가 이 유통사 — 잘 고르면 Track B 강점, 못 고르면 실행 품질 실패. 메콩 채널 역량 검증이 선결.

🟦 라이브케어: "메콩의 현지 채널은 저희 진출에 큰 강점입니다. 파트너십을 진지하게 논의하고 싶습니다. 서로 성공하려면 몇 가지를 함께 설계하고 싶어요 — ①메콩이 충분한 마진과 함께 팔 수 있는 가격·지원 구조 ②현지 농가 대상 AS·기술 교육 체계(저희가 초기 교육 지원) ③지역·기간·성과 기반 파트너십 범위요. 15~30분 미팅에서 메콩의 채널 커버리지와 협업 구조를 함께 보실까요?"
🟥 메콩: "좋습니다. 마진과 AS 지원, 저희 채널 보호하는 지역 보장이 되면 적극 밀 수 있습니다. 미팅에서 구조를 잡죠."

**■ 종료: 채널 파트너십 협의(conditional).** 채널 파트너 판단(팔 수 있는가)에 맞춰 마진·지원·지역 구조 설계, 현지 실행 역량은 검증 조건.

# 케이스 L3 — 베트남 정부·ODA 축산개발 프로그램 (Track C 공공) · in/sell/conditional

**🔒 봉인**: 농업농촌개발부 산하 스마트축산 시범사업(ODA 연계). 구매이유 = 국가 축산 생산성 향상 + 소농 지원(Impact) + 시범 성과(정책 실적) — 상업 매출 아님. Key Benefit = Impact+Problem Solving(상업의 Cost Reduction과 다름). 제약 = 공공 조달·ODA 절차(투명성·행정 복잡), 소농 지불능력 낮음(보조금·펀딩 구조), 시범 성과 측정 필요, 정치·행정 일정. 라이브케어 가치 = 매출 제한적이나 국가 레퍼런스 + ODA 실적 + 시장 교두보.

## 대화 전문 (context)

🟥 ODA 프로그램 담당: "라이브케어의 스마트축산 솔루션에 관심 있습니다. 저희 축산개발 시범사업에서 소규모 농가 생산성을 높이고 데이터 기반 관리를 실증하고자 하는데, 파트너로 참여할 의향이 있으신가요?"

🔎 [정합—Impact+PS]: L1(Cost Reduction)·L2(마진)와 또 다른 정합 기준 — 여기선 국가 생산성·소농 지원(Impact) + 데이터 관리 실증(Problem Solving)이 구매이유. 라이브케어 인터뷰의 "ODA 트랙 = Impact+PS"가 정확히 대응. 상업 매출로 판단하면 unfit, 정책 목표·레퍼런스로 판단하면 fit.
🔎 [사업단계—실증 vs 상용, 전략 역전]: 매출이 아니라 국가 레퍼런스+ODA 실적+교두보. 에이엔폴리 A2(EU 컨소시엄)와 같은 구조 — 단기 매출 약해도 전략가치로 진행(#01 약한 차원 전략적 역전). 단 소농 지불능력 낮아 보조금·펀딩 구조가 상업화 전제.
🔎 [실증—개발 성과 지표]: 공공/ODA 실증 신호 = 개발 성과(생산성 향상·소농 소득·데이터 실증)를 정책 보고 형태로. 키뮤 1-J(재활지표)·에이엔폴리 A2(탄소 KPI)와 같은 '공공 성과 지표' 계열.
🔎 [리스크—공공 조달·행정]: ODA 절차 투명성·행정 복잡성·정치 일정 의존이 선결·수익성 리스크. 스타트업 리소스 감당 + 지속가능성(시범 후 상업 전환)이 관건.

🟦 라이브케어: "이런 시범사업 참여는 저희에게 큰 의미입니다 — 소농 생산성을 데이터로 높이는 건 저희 미션과 정확히 맞고 베트남 축산 발전에 기여하고 싶습니다. 저희 솔루션의 생산성·소농 소득 효과를 시범사업 성과지표에 맞춰 설계하겠습니다. 다만 지속가능한 성공을 위해 시범 이후 상업 전환 경로와 소농 지불 구조(보조금·펀딩 연계)도 함께 논의하고 싶습니다. 15~30분 미팅에서 시범 범위와 성과 지표를 잡아볼까요?"

**■ 종료: ODA 시범사업 참여 검토(conditional).** Impact 정합 + 국가 레퍼런스 전략가치로 진행, 행정·지속가능성(상업 전환)이 조건.

## 3. 종합 — 이 도메인의 온톨로지 수확

- **한 판매자의 정합 기준 3분화**: 같은 체내 캡슐이 track에 따라 L1(Cost Reduction/문제해결)·L2(재판매 마진)·L3(Impact+정책성과)로 정합 기준이 완전히 갈림. 키뮤 "구매자 유형별 Impact 스펙트럼"이 축산 도메인에서 독립 재현.
- **대체재 비교가 딜을 지배한 유일 케이스(L1)**: 기존 대안(목걸이형)이 있는 시장에선 "신제품"이 아니라 "기존 대비 우위"가 핵심. 나란히 비교 PoC가 손잡이.
- **채널 긴장의 양면(L1↔L2 거울)**: 농장은 "직접거래 낯설다", 유통사는 "직접 진출 위협" — 같은 긴장의 반대편. 진입 채널은 연속 손잡이(코봇 C2와 함께).
- **현지 실행 파트너 검증(#04)**: Track B 성패를 가르는 선결조건.
- **공공 전략 역전 재확인(L3)**: 에이엔폴리 A2와 동일 구조 — 매출 약해도 레퍼런스 전략가치.
