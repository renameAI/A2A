# 케이스 C1 — 서밋 모빌리티 (미국 프리미엄 전동휠체어사) · out/sell/conditional

**🔒 봉인**: 구매이유 = 차기 모델 차별화(Permobil급 대비 밀림, 아웃도어·액티브 세그먼트 대응). 위험 = 컨셉 좋으면 자체 R&D로 유사 서스펜션 개발 검토(make-vs-buy 카피). 신뢰임계(선결) = FDA Class 분류·보험수가(HCPCS)·PL 보험 — 모듈 추가로 분류/보험 흔들리면 채택 불가. 원가 = 프리미엄 전가 가능하나 BOM 상한. 리스크 성향 = 신기술 조기채택 신중(리콜 치명적).

## 대화 전문 (context)

**R1** 🟦 코봇: "장애물 극복 휠 모듈 '미라클휠'을 만드는 코봇시스템입니다. 저희 모듈은 기존 휠체어 설계를 대폭 바꾸지 않고 장착만으로 계단·턱 극복 기능을 더해 프리미엄 라인 차별화를 만듭니다. CES 2025 혁신상, 일본 독점공급 중입니다. 15분 소개 가능할까요?"
🟥 서밋: "흥미롭네요 — 마침 차기 모델 차별화를 고민 중이라 컨셉 자체는 관심이 갑니다. 그런데 저희 업계 특성상 먼저 확인할 게 있어요. ①모듈 추가하면 FDA 분류가 바뀌나요? ②보험수가(HCPCS) 적용에 영향은? ③제품책임 이슈는? 저희에겐 기능보다 이 규제·보험 문제가 채택을 좌우합니다. 리콜 한 번이면 회사가 흔들리는 업계라서요."

**R2** 🟦 코봇: "핵심을 정확히 짚으셨습니다. 저희 모듈은 기존 기기의 FDA 분류를 유지하도록 설계됐습니다(장착 방식이 본질 기능을 안 바꿔서요) — 다만 기기 구성마다 다르므로 서밋 제품 기준 규제 영향 분석 자료를 만들어 드리겠습니다. 보험수가·PL도 일본·기존 적용 사례 데이터가 있습니다. 15분 미팅에서 규제 영향 분석·적용 사례를 함께 보시고, 관심 있으시면 소량 PoC로 자체 테스트하시는 방향은 어떨까요?"
🟥 서밋: "규제 영향 분석을 서밋 기준으로 만들어주시면 큰 도움입니다. PoC도 좋고요. 다만 솔직히 — 저희 R&D팀은 이런 걸 보면 '우리가 직접 만들 수 있나'도 검토하는 편이에요. 미팅에서 '왜 자체 개발이 아니라 코봇이어야 하는지'도 설득되면 좋겠습니다."

**R3** 🟦 코봇: "make-vs-buy 검토는 당연하고 건강한 과정입니다. 자체 개발 대비 저희 가치는 셋 — ①핵심 메커니즘이 특허 보호돼 유사 개발 시 회피 설계 부담·IP 리스크, ②일본 양산·CES 검증으로 개발 시간 수년과 실패 리스크 절감, ③규제 영향 분석·적용 데이터 제공. 자체 개발로 이 셋을 재현하는 비용과 비교해 보시면 좋겠습니다. 다음 주 15~30분 R&D·규제 담당과 함께 보시죠."
🟥 서밋: "합리적이네요. 특허 부분은 법무도 봐야겠고요. R&D·규제·법무 함께 목요일에 보죠. 규제 영향 분석 자료 미리 주시면 준비하겠습니다."

**■ 종료: 미팅 확정(conditional) — 카피 위험을 특허·시간가치로 방어.** 실제 채택은 규제 분석·PoC·make-vs-buy 결론에 달림.

## 차원별 매칭

| 차원 | 판정 | 근거 | 온톨로지 루브릭 |
|---|---|---|---|
| industry_fit | fit | 전동휠체어 모듈 판 일치 | |
| purpose_alignment | fit | 차기 모델 차별화 니즈에 부합 | 정합 강함 — 문제는 규제·카피 실행 리스크 |
| resource_complementarity | fit | 장애물 극복 모듈 ↔ 차별화 결핍 | |
| stage_compatibility | caution | 신모델 개발주기 2~3년, PoC 타이밍 | 모듈 도메인 단계 신호 = 완성차/기기 개발 주기 |
| demonstrability | **caution(선결)** | FDA 분류·보험수가·PL | ★★실증 신호 = **규제·보험 데이터(FDA Class·HCPCS·PL)**. 기능보다 이게 채택 좌우 |
| substitute_comparison *(buy)* | **caution(카피 위험)** | "자체 개발 가능?" make-vs-buy | ★★대체재가 **경쟁사가 아니라 구매자 자신의 R&D**. 방어 무기 = 특허·시간가치·검증. #03 카피 거절의 '정당한 버전' |
| 리스크 | precondition | 규제 영향, PL | deal-breaker 후보 = FDA 분류 변경 시 채택 불가 |

**리스크 3분류**: FDA 분류=선결(변경 시 결렬), 보험수가=선결, make-vs-buy=수익성(방어 가능), BOM 상승=기각(프리미엄 전가).
**결정**: `conditional` — 정합 강하나 규제(선결)·카피(대체재) 조건.
**키뮤 1-F(카피)와 대조**: 1-F는 착취적 의도→추격 중단, C1은 정당한 검토→특허·시간가치로 방어.

# 케이스 C2 — 유로케어 디스트리뷰션 (유럽 복지기기 유통사) · in/sell/conditional

**🔒 봉인**: CES 보고 먼저 연락 — 유럽 전역 독점 유통권 원함. 구매이유 = 차별화 신제품 포트폴리오 + 독점으로 마진·협상력. 핵심 충돌 = 전역 독점 요구 vs 코봇의 전략 자유도(일본 독점계약 조항의 확장 해석·향후 글로벌 파트너 구조와 충돌 소지). 대안 = 경쟁 모빌리티 신제품. 우려 = 코봇 직접 진출 시 유통사 존재이유 소멸 → 독점 요구 이유.

## 대화 전문 (context)

🟥 유로케어: "CES에서 미라클휠 보고 연락드립니다. 유럽 복지기기 시장에서 저희가 강한 채널을 갖고 있습니다. 저희에게 유럽 전역 독점 유통권을 주시면 대대적으로 밀어드리겠습니다."

🔎 [진위 게이트—통과, but 조건 검토]: 실재 유통사·공식 연락. 진위 통과. 인바운드+Willingness 높음(독점 원함). #04·키뮤 1-G 교훈 — 높은 Willingness가 함정. 독점권은 상대엔 매력이나 코봇엔 전략 자유도를 묶는 리스크.
🔎 [정합—채널 전략]: 코봇 미국 전략 = 직접(모듈 PoC→공동개발). 유럽은 유통사 경유가 빠르나 전역 독점을 주면 향후 직접 진출·타 파트너 선택지 상실. 유로케어가 원하는 것(독점)과 코봇에 필요한 것(유연성) 충돌.
🔎 [대체재/기회비용—코봇 관점]: 유로케어 독점 수락 시 다른 유럽 채널·직접 진출 옵션 포기(기회비용 큼). 단 유통사 빠른 접근은 매력. → 독점 주되 범위 제한 재설계.
🔎 [독점계약 충돌—선결]: 일본 독점계약 조항이 유럽 딜에 영향 없는지 법무 확인(선결). 유럽 독점 준다면 지역·기간·성과 조건부로 한정해 자유도 보존.

🟦 코봇: "관심 감사합니다. 유로케어의 채널은 큰 강점이 될 수 있습니다. 다만 처음부터 전역 독점보다는 — 특정 국가(독일·오스트리아)에서 기간·성과 조건부 독점으로 시작하고, 성과 나오면 지역 확대하는 구조를 제안합니다. 이러면 유로케어도 리스크 없이 시작하고 저희도 시장을 함께 키웁니다. 전역 독점은 초기부터 양측에 부담이 클 수 있어서요. 15~30분 통화로 구조를 함께 잡아볼까요?"
🟥 유로케어: "전역 독점을 못 받으면 투자 유인이 줄긴 합니다. 하지만 성과 조건부로 확대되는 구조라면 얘기해볼 만하네요. 독일·오스트리아로 시작해 성과 나면 확대, 그 조건을 구체화하죠."

**■ 종료: 채널 재설계 후 조건부 독점 협의(conditional).** 전역 독점 → 지역·기간·성과 조건부 축소. Willingness 함정(독점)을 범위 제한으로 관리.

## 3. 종합 — 이 도메인의 온톨로지 수확

- **실증 신호 추가**: 규제·보험 데이터(FDA·HCPCS·PL) — 기능보다 규제가 채택 좌우.
- **대체재의 새 형태 2종**: C1 = 구매자 자신의 R&D(make-vs-buy) / C2 = 판매자 자신의 기회비용(독점 수락 대가).
- **카피 대응 분기 확정**: 키뮤 1-F(착취적→추격 중단) vs 코봇 C1(정당한 검토→특허·시간가치 방어) — 같은 '카피'가 의도에 따라 대응 정반대.
- **진입 채널 = 연속 손잡이**: 직접/유통 이분법이 아니라 지역·기간·성과로 재설계.
- **기존 계약 충돌**: 새 선결조건 유형(독점계약의 지역 확장 해석).
