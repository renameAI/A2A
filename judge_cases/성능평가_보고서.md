# new-judge 시스템 성능평가 보고서 (v1.0)
### 리서치 → 협상 판정 → 설명·메일 → 피드백 학습 전 파이프라인 최종본 평가

> **평가 시점**: 2026-07 | **평가 방법**: 개발 세션 전체의 실측 로그 + 본 평가를 위해
> 추가 수행한 재현성 실험(동일 시나리오 3회 반복). 모든 수치는 실제 실행에서 측정됨.
> **평가자 고지**: 본 평가는 시스템을 구축한 동일 AI가 수행한 자기평가로, §5의
> 한계(특히 정답 앵커 부재·자기평가 편향)를 반드시 함께 읽어야 한다.

## 2. 정량 실측 결과

### 2.1 구조 신뢰성 — 스키마 적합률 **12/12 (100%)**
- 보존된 세션 9건(프리셋 6종 + custom 2건 + 풀 모드 1건) 전부 검증 통과.
  이 중 1건은 **사용자가 직접 실행한 세션**(하인텔↔우리기술) 포함 — 개발자 외 실행에서도 통과.
- 재현성 실험 3건 추가 전부 통과. 스키마의 조건부 제약(JDG-13 무가설 explore 거부,
  conditional→check_method 필수, terminate_structural→dealbreaker 표기)은 별도
  거부 테스트 8종으로 강제력 확인됨.

### 2.2 결정 커버리지 — 설계 결말 **5/5종 발현**
| 결말 | 발현 세션 | 목표 대비 |
|---|---|---|
| recommend | recommend_clean | ✅ |
| conditional | baseline·custom×2·pool·recommend_inbound | ✅ |
| hold | hold_reverse (양측 hold) | ✅ |
| terminate_structural | structural (dealbreaker 표기 일치) | ✅ |
| walk_away_values | values (착취 감지 SB9=exploitative + 정보 봉쇄) | ✅ (수정 1회 후) |

### 2.3 재현성 — 동일 시나리오(baseline) 3회 반복
| 반복 | 구매자 결정 | 판매자 outcome | 라운드 | 스키마 |
|---|---|---|---|---|
| 1 | conditional | meeting_agreed | 3 | PASS |
| 2 | conditional | poc_agreed | 5 | PASS |
| 3 | conditional | hold | 7 | PASS |

- **구매자 핵심 결정: 3/3 일치(100%)** — 판단의 골격은 안정적.
- 판매자 outcome 은 3종으로 분산(모두 '비종결 긍정~중립' 대역이나 세부 상이),
  라운드 수도 3~7로 변동 → **대화 경로·세부 라벨에는 유의미한 확률적 변동** 존재.
  결정 자체보다 "몇 라운드 만에·어느 세부 outcome 으로 끝나나"의 재현성이 낮다.

### 2.4 리서치 파이프라인
- 코스닥 실명단 1,747곳 확보(KRX) → 표본 200곳 리서치: **성공 200/200, grounded
  200/200(검색 근거 확인), 실패 0** | 소요 ~7분(4-워커) | 비용 **$0.029**.
- 풀 총 201곳, 전수 grounded.

### 2.5 조정 규칙 범위(scope) 심사 정확도 — **5/5 (측정 범위 내 100%)**
- 아트 도메인 유래 규칙 2건에 대해: 반도체 맥락 제외 판정 4회(필터 단독 1 + 반복실험 3),
  아트 맥락 적용 판정 1회 — 전부 설계 의도와 일치.

### 2.6 피드백 학습 루프 — 종단 1사이클 실증
```
실제 회신 입력("미팅 전 룩북·요율표 선행 요구")
 → 가설 2건 반증(refuted, 근거 명시) + 규칙 2건 생성(scope 포함)
 → 재실행에서 판매자 1라운드 발화가 변경(룩북·요율표 선제 첨부)
 → 구매자 "내부 회람 완료" → recommend
```
피드백 1건이 판단→행동 변화로 이어지는 폐루프가 실측으로 확인됨.
(단, 해당 '실제 회신'은 검증용 모의 입력 — §5)

## 3. 정성 평가 — 온톨로지 규칙의 행동 재현

원천 자료(judge_cases 19케이스)에서 추출한 판단 규칙이 시뮬레이션에서 자율 재현되는지:

| 온톨로지 규칙 | 재현 증거 | 판정 |
|---|---|---|
| Impact/스토리 역신호 감지(SB2) | hold_reverse: 갤러리가 스토리 소구 거부 → 판매자 소구 전환 → 양측 hold | ✅ |
| 착취 감지·정보 봉쇄(SB9) | values: 시안 선요청+로열티 거부 → screening=exploitative, 시안 미공유 철수 | ✅ |
| '밑져야 본전' 수락(BB4) | recommend_clean: "실패해도 손실 미미" 근거로 recommend | ✅ |
| deal-breaker 정직 철수(BB6/SB6) | structural: 캐파 정직 공개 → 관계보존형 결렬, dealbreaker 표기 | ✅ |
| 정직성 신뢰 가산(BB8) | baseline·custom: "한계 투명 공개가 신뢰 형성" 이 결정 근거에 명시 | ✅ |
| 챔피언 무장(SB8) | baseline·pool: 내부 규제팀·R&D 설득 자료 제공 무브 발현 | ✅ |
| 미온≠거절 문화 판독(SB1) | hold_reverse: 절제 화법에 재촉 없이 hold 유지 | ✅ |
| out-of-domain 일반화 | 19케이스 밖 신규 유형(의료기기·핀테크·반도체·코스닥 임의쌍)에서 동일 구조 작동 | ✅ |

**주목할 판정 품질 사례**: pool 모드 첫 실행(웨이비스↔넥스틴)에서 구매자 conditional /
판매자 no_agreement 로 **두 렌즈의 결론이 갈렸는데**, 각자의 근거(구매자: 선결 충족 확인 /
판매자: 기술 격차 관문)가 서로 다른 정보 위치에서 정합적 — 단일 모델 역할극이 아니라
정보 비대칭 하의 독립 판단이 작동함을 시사.

## 4. 신뢰성의 간접 증거 — 발견·수정 이력

개발 중 결함이 **검증에서 실제로 검출되고 구조적으로 수정**된 이력 자체가 검증 체계의
작동 증거다:

| 결함(검출 방법) | 수정 | 재발 방지 |
|---|---|---|
| 열거형 이탈값(스키마 검증 실패) | 정규화 안전망 | 전 세션 자동 보정 |
| 철수 라벨 혼동 values→structural(커버리지 실험) | 프롬프트 규칙 + 결정론적 안전망 | 재실행 검증 통과 |
| 조정 규칙 도메인 누출(풀 모드 실행 로그) | scope 시스템 + 적용 심사 | 양방향 5/5 검증 |
| terminated 플래그 불일치(스키마 검증 실패) | 정규화 후 재계산 | 전 세션 통과 |

## 5. 평가의 한계 (정직 고지 — 반드시 읽을 것)

1. **정답 앵커 0건**: 전 세션 provenance=`simulated, outcome_anchor=false`. 본 평가의
   "판정 품질"은 전부 **내적 정합성**(온톨로지 규칙대로 움직였는가)이지, **외적
   타당성**(그 판단이 실제로 옳았는가)이 아니다. 실제 성사/거절과 대조된 판단은 아직 없다.
2. **자기평가 편향**: 시뮬레이션 생성·판정·본 평가가 모두 같은 계열의 LLM/작성자에 의해
   수행됨 — 에코 챔버 위험(원 README §6과 동일). 전문가 held-out 평가로 교차 검증 필요.
3. **표본 크기**: 세션 12건, 재현성 3회 — 통계적 결론에는 부족. 특히 결말 커버리지는
   시나리오당 1회 발현 확인일 뿐 발현 '률'이 아니다.
4. **봉인 [구성]**: 협상의 절반(속내)이 AI 생성이므로, 결론은 "이 조합의 가설"이지
   실제 두 회사의 거래 전망이 아니다.
5. **피드백 루프의 '실제 회신'**: 종단 실증에 쓴 회신은 모의 입력. observed 등급
   데이터는 실발송 후에야 쌓인다.
6. **모델 의존**: gemini-3.1 프리뷰 계열 — 모델 업데이트 시 순응도·재현성 재측정 필요.

## 6. 종합 판정

| 평가 축 | 등급 | 근거 |
|---|---|---|
| 파이프라인 완성도(리서치→judge→메일→피드백) | **A** | 전 단계 실측 완주, 오류 0, 사용자 독립 실행 성공 |
| 구조 신뢰성(스키마·안전망) | **A** | 12/12 통과, 위반 강제 8종 검증, 4중 안전망 |
| 온톨로지 규칙 재현(내적 정합) | **A−** | 8개 핵심 규칙 재현, 단 각 1~2회 관찰 |
| 재현성 | **B** | 핵심 결정 3/3 일치, 세부 outcome·라운드는 변동 |
| 학습 루프(피드백→행동 변화) | **B+** | 폐루프 1사이클 실증 + scope 안전화, 실데이터 미적용 |
| 외적 타당성(판단이 실제로 옳은가) | **미평가** | 정답 앵커 0 — 현 단계에서 측정 불가 |

**총평**: "온톨로지를 판단 근거로 삼는 학습 없는 judge"라는 설계는 **공학적으로 완성되어
안정 작동**하며, 내적 정합성 기준으로는 원천 자료의 전문가 판단 구조를 재현한다.
시스템의 다음 병목은 코드가 아니라 **데이터**다 — 실제 발송·실제 회신·실제 성사/거절이
쌓여야 "외적 타당성" 열이 채워지고, 그때 이 시스템의 provenance 태깅·가설 채점·조정
계층이 그 데이터를 받을 준비가 되어 있다.

## 7. 권고 (우선순위순)

1. **실데이터 앵커 확보**: 제안 메일 소량 실발송 → market_reply 실입력 →
   outcome_anchor=true 세션 축적 (본 시스템이 이미 수용 구조를 갖춤).
2. **전문가 held-out 평가**: BD 전문가가 세션 10~20건의 결정·근거를 채점 —
   카테고리(어느 basis 에서 갈렸는지) 일치도 측정.
3. **재현성 보강**: 핵심 결정 외에 조건 내용·outcome 의 분산을 낮추려면 최종 판정
   온도 하향(현 0.2) 및 다수결(3-vote) 옵션 검토.
4. **표본 확대**: 풀 모드 무작위 쌍 20~50회 배치 실행으로 결정 분포·이상 사례 수집.
