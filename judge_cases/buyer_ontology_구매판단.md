# BUYER 판단 온톨로지 — 구매자 시야의 판단 채널(basis)과 핵심가치·검증질문
### judge_cases 19케이스(5개사 × 5도메인) 교차 분석 산출물 (v1.0)

> **용도**: 구매자 역할 AI의 대화 판단 근거. "상대 회사의 제품을 살 것인가"를 결정하기
> 위해, 현재까지의 대화 문맥을 본 온톨로지의 각 basis에 비추어 검토하고, 미충족
> basis를 채우는 질문으로 다음 발화를 선택한다.
> **출처**: kimu(10) · anpoly(2) · cobot(2) · mushn(2) · livecare(3) 케이스의
> 구매측 발화·추론 궤적·차원별 매칭표를 구매자 독립 시야로 재분석.
> **⚠️ provenance**: 원 재료의 구매측은 전부 `[구성]`(시뮬레이션)이다. 본 온톨로지는
> 판단 **구조**의 추출이며, 각 basis의 도메인별 임계값은 실데이터로 보정 대상이다.

---

## 0. 구매자 판단의 대원칙 (19케이스 교차 불변량)

1. **내가 사는 가치 ≠ 상대가 파는 가치.** 판매자의 자랑 포인트(친환경·Impact·기술)는
   내 구매 이유가 아닐 수 있다 — 전제이거나(A1), 보너스이거나(케이스1), 역신호일 수
   있다(1-C). 판단 기준은 항상 "**나는 이것으로 무엇을 하려는가**".
2. **산업 fit은 필요조건이지 충분조건이 아니다.** 판이 맞아도 실행 게이트 하나로
   결렬된다(1-E).
3. **높은 열의(Willingness)는 판단이 아니다.** 상대가 간절해도, 내가 감동했어도,
   실질(정합·단계·실행)을 따로 판정한다(1-G).
4. **미온 ≠ 거절 ≠ 부적합.** 내 미온이 방향 문제인지 증거 문제인지 문화적 절제인지
   구분해 표현한다(1-I).
5. **결정은 점수가 아니라 4종이다**: `recommend / conditional / hold / terminate`
   — 근거·부족·리스크를 남기는 구조화 판단.

## 1. Basis 요약 지도 — 구매자의 10개 판단 채널

| # | basis (판단 채널) | 한 줄 질문 | 주 근거 케이스 |
|---|---|---|---|
| BB1 | 목적 정합 | 이게 **내 목적**(과 내 고객의 목적)에 꽂히는가 | 전 케이스, 특히 1·1-B·A1·L2 |
| BB2 | 가치 서열 | 상대의 강조점이 **내 1순위 구매 이유**와 일치하는가 | 1-C·1-D·A1·A2·L1~L3 |
| BB3 | 대체재 우위 | 기존 대안(현상유지·경쟁사·**자체 개발**) 대비 나은가 | L1(지배)·C1·1-H·A1·M1 |
| BB4 | 기회비용·다운사이드 | 수용하면 무엇을 포기하고, 실패하면 무엇을 잃나 | 1-H·1-J·M2 |
| BB5 | 실증 신호 | 주장이 **내 도메인의 신뢰 화폐**로 증명되는가 | 전 케이스(도메인별 상이) |
| BB6 | 실행·선결 게이트 | 규제·캐파·공급·AS를 실제로 감당하는가 | 1-E·C1·M1·L1 |
| BB7 | 단계·타이밍 | 내 캘린더·사이클의 **지금**에 맞는가 | 1·1-B·1-C·C1·M2 |
| BB8 | 신뢰·진정성 | 실재하는가, 진짜인가, 정직한가 | 1-G·1-B·1-D·1-E·1-I |
| BB9 | 결정 구조 | **내가 결정자인가**, 내부 승인은 어떻게 통과하나 | 1-D·A1·C1·1-I |
| BB10 | 계약·정보 보호 | 권리·비밀·계약 구조가 깨끗하고 수용 가능한가 | 1-B·M1·M2·C2·1-E |

각 basis는 `unknown → assumed → confirmed` 상태를 가지며, 판정은
`fit / caution / unfit` + 근거 1문장으로 남긴다.

## 2. Basis별 상세 — 핵심가치 · 하위신호 · 검증질문

### BB1. 목적 정합 (Purpose Fit)

**핵심가치**: 상대의 제안이 "내가 이것으로 이루려는 것"에 직접 기여하는가.
내 목적은 종종 **나의 고객("구매자의 구매자")** 까지 이어진다.

**구매자 유형별 목적의 이동** (19케이스 실증):
```
최종소비자 판매(브랜드: 매대 매력) → 내 클라이언트 피칭(대행사: 현재 캠페인 적합)
→ 미적 판단(갤러리: 큐레토리얼 정당성) → 조직 KPI(CSR: ESG 목표)
→ 사회적 미션(복지센터·ODA: 수혜자 성과) → 연구 성과(연구소: 논문·펀딩)
→ 재판매 수익(유통사·컴파운더: 내 고객 OEM/농장이 사는가 + 내 마진)
```

**하위신호**: 상대가 내 사용 맥락을 이해하고 말하는가 / 내 고객의 요구까지 짚는가 /
범용 제안("무엇이든 가능")이 아니라 내 상황에 맞춘 제안인가.

**검증질문(대화에서 상대에게)**:
- "우리가 이걸 도입해 달성하려는 것은 [X]입니다. 그 관점에서 이 제안이 정확히
  어떻게 기여합니까?"
- (중간자일 때) "이건 결국 우리 고객[OEM/클라이언트/농장]이 사야 성립합니다.
  그들에게 뭐라고 말할 수 있게 해줍니까?"
- "특정 [캠페인/모델/프로젝트]를 염두에 두었습니까, 범용 제안입니까?" (1-B)

**판정 기준**: fit = 내 목적의 언어로 구체 기여 확인 / caution = 방향은 맞으나
내 맥락 미반영 / unfit = 방향 자체가 다름(1-E: 회전율 ↔ 고부가 소량) 또는
가치 충돌(1-F).

### BB2. 가치 서열 (Value Hierarchy — 실구매 이유 1순위)

**핵심가치**: 내 돈을 내게 만드는 **1순위 이유**가 충족되는가. 부가 가치는 1순위
충족 후에만 의미가 있다.

**하위신호 — 가치 스펙트럼 위치 판정** (같은 자산이 유형마다 다른 위치):
```
역신호 ←──────────────────────────────────────────→ 1순위·미션
갤러리(앞세우면 거부) — 컴파운더(전제일 뿐) — 브랜드(보너스) — 대행사(차별화 무기)
— CSR(진짜 구매이유) — 복지센터·ODA(미션 자체)
```
- 판매자가 자랑하는 가치가 내게는 **전제**일 수 있다(A1: "친환경은 세일즈 포인트가
  아니라 전제 — 내가 보는 건 물성").
- 내 서열: 예) 브랜드 = 디자인 신선도 55% > 가격 30% > CSR 15%(케이스1 봉인).

**검증질문**:
- "우리에게 결정적인 건 [1순위]입니다. 그 기준으로 당신 제안의 답은 무엇입니까?"
- "그 [부가가치]는 좋습니다만, [1순위]가 안 되면 무의미합니다. [1순위]부터 봅시다."
- (자기점검) "상대의 강조점에 내 서열이 흔들리고 있지 않은가?"

**판정 기준**: fit = 1순위 직접 충족 / caution = 부가가치만 강조하고 1순위 미확인 /
unfit = 1순위와 무관하거나 역신호 자산을 계속 앞세움.

### BB3. 대체재 우위 (Substitute Comparison)

**핵심가치**: 이미 있는 대안 대비, **전환 비용을 넘는 우위**가 있는가.
19케이스에서 대체재는 5형태로 나타났다:

| 대체재 형태 | 케이스 | 판정 축 |
|---|---|---|
| 풍부한 기존 공급 풀 | 1·1-H(유럽 아티스트), 1-C(월 수십 포트폴리오) | 굳이 이 상대여야 하는 이유 |
| 사내 자원 | 1-B(자체 아트팀) | 내부로 대체 불가한 요소 |
| **자체 개발(make-vs-buy)** | C1(자체 R&D 검토) | 특허 회피 비용·개발 시간·실패 위험 |
| 경쟁 공급사 | A1(핀란드사)·M1(타 균사체사) | 데이터로 나란히 비교 |
| **기존 도입 대안(현상 유지)** | L1(목걸이형 — **딜 전체를 지배**) | 전환 부담을 넘는 증분 가치 |

**검증질문**:
- "이미 [기존 대안]을 쓰고 있습니다. 그것 대비 무엇이, 어느 정도 낫습니까?"
- "우리가 직접 만들면 안 됩니까? 특허·시간·리스크 관점에서 설명해 보세요." (C1)
- "기존 것과 **나란히 비교**할 수 있는 방법(비교 PoC)을 제안할 수 있습니까?" (L1)
- "경쟁사 [X]와도 이야기 중입니다. 그쪽 대비 차별점은?" (A1)

**판정 기준**: fit = 대체 불가 요소 입증(실재성·특허·데이터 우위) / caution =
차별점 주장만 있고 검증 미완 / unfit = 기존 대안으로 충분.

### BB4. 기회비용·다운사이드 (Opportunity Cost & Downside)

**핵심가치**: 수용 시 포기하는 것(슬롯·예산·리소스)과 실패 시 손실이 감수 가능한가.
**다운사이드가 낮으면 "밑져야 본전"으로 수락 가능**(1-H: MG 제거로 손실 0 →
미팅 수락의 실제 이유).

**하위신호**: 희소 슬롯(시즌·전시) 소모 여부 / 최소보장(MG)·선투자 요구 여부 /
최소 단위 시험 가능 여부 / 성공 시 업사이드(1-J: 성과→재원 확대, M2: 레퍼런스).

**검증질문**:
- "이걸 받으면 나는 무엇을 못 하게 되는가?" (자기질문)
- "실패했을 때 우리 손실은 정확히 무엇입니까? 최소보장을 요구합니까?"
- "가장 작은 단위(소수 개체·직영 3곳·8주 PoC)로 먼저 시험할 수 있습니까?"
- "성공하면 무엇이 커집니까(재원·레퍼런스·확장)?"

**판정 기준**: fit = 다운사이드 낮음+업사이드 확인 / caution = 슬롯 희소하나 축소
가능 / unfit = 선투자·독점 등으로 손실 구조가 큼.

---

### BB5. 실증 신호 (Demonstrability — 내 도메인의 신뢰 화폐)

**핵심가치**: 주장이 아니라 **내 도메인이 신뢰하는 화폐**로 증명되는가.
"보편 차원 + 도메인별 신호"의 2층 구조:

| 내 도메인 | 내가 요구할 신뢰 화폐 | 케이스 |
|---|---|---|
| 소비재/리빙 | 실물 샘플 품질 | 1 |
| 대행사/콘텐츠 | IP·상업 사용권 클리어 문서 | 1-B |
| 갤러리/문화 | 담론 내 위치("왜 지금 이 작가") | 1-C |
| CSR/ESG | 수혜자 수·배분 투명성(워싱 방어 숫자) | 1-D |
| 복지/교육 | 재활·교육 성과 지표 | 1-J |
| 소재/부품 | 등급 물성 데이터(열안정성·VOC·내후성) | A1 |
| 공공 실증 | KPI 정량화(탄소·생산성·소득) | A2·L3 |
| 의료기기 | 규제·보험 데이터(FDA Class·수가·PL) | C1 |
| 식품 원료 | 식품안전 규제 상태(Novel Food) | M1 |
| 연구 | peer-review 가능성(연구가 실증을 '생산') | M2 |
| 현장 도입 | 현지 검증 레퍼런스 + 나란히 비교 데이터 | L1 |

**검증질문**:
- "[내 화폐]에 해당하는 데이터/문서/실물이 지금 있습니까, 언제 가능합니까?"
- "그 증거를 **미팅 전에** 보내줄 수 있습니까?" (A1: 데이터 시트 사전 검토)
- "직접 검증할 방법(샘플 자체 평가·비교 PoC)을 제공합니까?"

**판정 기준**: fit = 화폐로 증명+자체 검증 경로 제공 / caution = 증거 준비 중
(해소 경로 합의 시 진행 가능) / unfit·게이트 = 선결형 증거(규제 등) 부재 시
미팅조차 무의미(A1).

### BB6. 실행·선결 게이트 (Execution & Preconditions)

**핵심가치**: 상대가 이 거래를 **실제로 감당**하는가. 여기서의 미달은 소구로 풀리지
않는 **deal-breaker**다(1-E).

**하위신호**: 인증·규제 적합(FDA 분류 유지 여부, Prop65/CPSIA, Novel Food 일정) /
양산·공급 캐파(500매장 초도·리오더, PB 물량) / 캐파 확장 경로(CMO — M1은 이것으로
살았고 1-E는 없어서 죽었다) / 배상책임보험 / AS·현지 대응 체계(L1·L2) /
현지화(언어·강사 — 1-J 선결) / 기존 계약과의 충돌(C2: 일본 독점 조항).

**검증질문**:
- "미팅 전에 서류로 확인합니다 — 인증·캐파·보험, 됩니까?" (1-E 방식)
- "파일럿은 되는데 본 물량은 어떻게 감당합니까? 확장 경로(CMO 등)가 있습니까?"
- "문제가 생기면 누가, 어디서, 얼마 만에 대응합니까?"
- "당신의 기존 계약(독점 등)이 이 거래와 충돌하지 않습니까?"

**판정 기준**: fit = 게이트 전부 통과 또는 신뢰 가능한 해소 경로 / caution =
시간·비용으로 확보 가능 / **unfit(deal-breaker) = 구조적 미달 → terminate.
단, 정직하게 인정한 상대에겐 관계 보존**("규모 되면 재접촉", 1-E).

---

### BB7. 단계·타이밍 호환 (Stage & Timing)

**핵심가치**: 내 캘린더·사이클의 "지금"에 맞물리는가. 타이밍은 도메인마다 다른
시계로 돈다:

| 시계 유형 | 예 | 부호 |
|---|---|---|
| 시즌 캘린더 | SS 4개월 전(1), 전시 6~12개월 전(1-C) | 제약 |
| 진행 중 데드라인 | 피칭 마감(1-B) | **촉매**(같은 차원이 정반대 부호) |
| 개발·인증 주기 | 신모델 2~3년(C1), 샘플 6개월+OEM 승인 수년(A1) | 장기 제약 |
| 예산·승인 주기 | CSR 연간 기획(1-D), 학기·펀딩(M2), 품의(1-I) | 절차 제약 |

**검증질문**:
- "우리 [시즌/개발/예산] 사이클상 지금 들어오면 [X]까지 가능합니다. 그 일정을
  감당합니까?"
- "이 미팅이 **딜의 초입**임을 이해합니까? (평가 6개월, 승인은 그 뒤)" (A1)
- "당신의 [규제 승인] 일정이 우리 상업화 계획과 맞물립니까?" (M1)

**판정 기준**: fit = 사이클 맞물림(또는 데드라인 촉매) / caution = 어긋나나 다음
사이클로 이월 가능 / unfit = 사이클 자체가 못 만남.

---

### BB8. 신뢰·진정성 (Trust & Authenticity)

**핵심가치**: 상대가 실재하고, 주장하는 가치가 구조로 뒷받침되며, 한계를 정직하게
말하는가.

**하위신호**:
- **진위 게이트**(인바운드 선행): 실재 기업·공식 채널·검증 가능 이력. **통과는 끝이
  아니라 시작**(1-G) — 이후 실질 판정으로.
- **워싱·토크니즘 방어 구조**: 선언이 아니라 구조(당사자 크레딧·로열티 배분·투명
  공개)를 요구(1-B·1-D). "착한 척"은 역효과 리스크.
- **정직성 신호**: 한계(캐파·미확보 데이터)를 먼저 인정하는 상대는 신뢰 가산
  (1-E: "솔직히 말씀 감사합니다", A1: "일부 확보/시험 중" 투명 공개).
- **문화 코드**: 내 절제된 표현이 거절이 아님을 아는가, 재촉하지 않는가(1-I).

**검증질문**:
- "그 [사회적 가치/효과] 주장을 구조로 보여주세요 — 누가 어떻게 보상받고, 무엇이
  공개됩니까?"
- "현재 **안 되는 것**은 무엇입니까?" (정직성 테스트)
- (인바운드) 공식 채널·실재성 확인 후: "구체 실행 계획과 담당자가 있습니까?" (1-G)

**판정 기준**: fit = 구조로 증명+정직 / caution = 주장뿐, 구조 미확인 / unfit =
진위 미달, 또는 착취적 의도 감지.

---

### BB9. 결정 구조·내부 승인 (Decision Structure)

**핵심가치**: 이 거래를 실제로 승인하는 사람은 누구이며, 내부 관문을 통과할 수
있는가. **나(담당자)는 종종 결정권자가 아니다**(1-D).

**하위신호**: 승인 단계 수(마케팅·경영진, R&D·규제·법무 동석 — C1) / 부서 분리
(구매 ≠ R&D — A1) / 내부 설득에 필요한 것(측정 성과·미디어 앵글·예산 소관 정리 —
1-D) / 품의 문화의 속도(1-I).

**검증질문(상대에게 + 자기에게)**:
- (자기) "내가 좋다고 판단해도, 누구를 더 설득해야 하나? 무엇이 있으면 통과되나?"
- (상대에게) "내부 보고용으로 [수치·구조·앵글]을 자료로 만들어 줄 수 있습니까?"
- "다음 미팅에 [R&D/규제/법무]를 부르겠습니다. 그들이 물을 것에 준비돼 있습니까?"

**판정 기준**: fit = 승인 경로 파악+통과 자료 확보 / caution = 다단계·장기 /
unfit = 통과 불가 구조(예산 소관 침범 등).

---

### BB10. 계약·정보 보호 (Contract & Protection)

**핵심가치**: 권리 관계가 깨끗하고, 정보 교환 순서가 안전하며, 계약 구조가 수용
가능한가.

**하위신호**: IP·상업 사용권 클리어(1-B) / NDA 순서 — 내 배합·공정을 열기 전 상호
NDA(M1) / 발표권 충돌의 분리 해소(공개 범위 vs 비밀 유지 — M2) / 독점의 범위·기간·
성과 조건(C2) / 표준 계약(벤더 등록·EDI) 수용 여부(1-E) / 로열티·크레딧 구조.

**검증질문**:
- "저작권·사용권이 완전히 클리어됩니까? 문서로 확인 가능합니까?"
- "상세 논의 전에 상호 NDA부터 체결합시다. 동의합니까?"
- "독점을 원한다면 — 지역·기간·성과 조건은 무엇입니까?"
- "우리 표준 벤더 절차(서류 선행)를 따를 수 있습니까?"

**판정 기준**: fit = 구조 합의 / caution = 조율 필요(순서·범위) / unfit = 권리
불투명, 착취적 구조 요구.

## 3. 구매자 결정 규칙 — basis 판정 → 4종 결정

| 결정 | 발현 조건 (케이스 실증) |
|---|---|
| `recommend` | 대부분 fit + 남은 caution이 확인 리스크로 관리 가능. **특히 BB4 다운사이드가 낮으면 caution이 남아도 진행**("밑져야 본전", 1-H) |
| `conditional` | 핵심 fit + 선결조건 명시(규제 확인·비교 PoC·현지화·규모 축소). 조건과 검증 방법을 반드시 문장으로 남김 (1-C·1-G·1-J·A1·C1·M1·L1~L3) |
| `hold` | 관심 있으나 확정 불가(조직 속도·문화·미확인 정보). **hold는 실패가 아니라 유효한 판단** — 자료·샘플로 확인 리스크를 채우며 관계 유지 (1-I) |
| `terminate` ① | 구조적 불가(BB6 deal-breaker): 상대가 요건을 못 갖춤 → 결렬하되 **관계 보존**, 조건 변화 시 재접촉 (1-E) |
| `terminate` ② | 가치충돌·착취(BB8): 상대 의도가 착취적(카피·무단 재해석) → **관계를 맺지 않고 정보도 주지 않음** (1-F) |

**차원 간 우선순위**: BB6(게이트) unfit → 즉시 terminate①, 나머지 판정 불요.
BB8 착취 감지 → 즉시 terminate②. 그 외에는 BB1·BB2·BB3(사는 이유)가 주도하고
BB4·BB5가 진행 형태(규모·조건)를 정한다.

## 4. 대화 AI 사용 프로토콜 (판단-갭-발화 루프)

```
[매 턴]
1. 대조: 현재까지 대화를 BB1~BB10에 비추어 각 basis를
   unknown / assumed / confirmed + fit / caution / unfit 으로 갱신.
2. 게이트 우선: BB6 unfit 증거 또는 BB8 착취 신호 → 종결 발화(유형①/② 구분).
3. 갭 선정: 미확정 basis 중 우선순위 최상 1개 선택.
   우선순위 = 게이트(BB6·BB8 선결형) > 사는 이유(BB1·BB2·BB3) >
   진행 형태(BB4·BB5·BB7) > 절차(BB9·BB10).
4. 발화 생성: 해당 basis의 검증질문 형(§2)을 대화 문맥에 맞춰 1개만 던진다.
   상대 주장을 내 1순위 언어로 되받아 확인한다("우리 기준은 X인데, 그 답은?").
5. 판정 갱신: 상대 답변이 basis를 채우면 confirmed로, 회피·모호하면 같은 basis를
   형태를 바꿔 재질문(최대 2~3회). 초과 시 그 basis를 리스크로 남기고
   conditional/hold로 결정 형태를 낮춘다.
6. 종료: 결정 4종 중 하나 + 근거·조건·확인 리스크를 문장으로 산출.
```

## 5. 한계와 보정 계획
- 구매측 원 데이터가 `[구성]`이므로, basis 구조는 신뢰하되 **도메인별 임계값**
  (예: 물성 항목, 수가 코드, MOQ 규모)은 실데이터·전문가 검증으로 교체할 것.
- 본 온톨로지는 결정 4종·리스크 3분류(선결/수익성/기각) 체계와 호환되며, 각 basis
  판정은 해당 스키마 필드(`category_judgments`)에 사상 가능.
